HotkeyHandler

UMLClass
UCR :: Class Diagram :: HotkeyHandler

Description

Handles binding of hotkeys and enforces repeat supression. Will also simulate events not supported by native AHK (eg Up events for joystick buttons)

Attributes

Visibility Name Type Description
public RegisteredBindings Hotkey

An associative array of all the bindings that are currently registered with the system. Used to allow disabling / enabling of all hotkeys (eg before / after bind mode) and to disallow conflicting hotkeys. Seeing as the name of a hotkey is only unique to a plugin, hotkeys are indexed by the HWND of their GuiControl, to provide a simpler, flatter structure and speed searches.

Operations

Visibility Name Description
public SetBinding(hk: Hotkey)

Sets the binding for a hotkey. Does not actually enable the hotkey, just registers it with the system.
public IsBindable(hk: Hotkey, bo: BindObject)

Returns true if the passed binding is valid (eg does not conflict with another binding, false if not.
public ChangeHotkeyState(state: bool, hk: Hotkey)

Turns on or off all registered hotkeys. If passed a hotkey as the 2nd parameter, only turns on or off that hotkey.
public KeyEvent(hk: Hotkey, event: bool)

The hotkey that an End-User bound to a hotkey instance was pressed or released.
