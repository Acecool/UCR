Plugin

UMLClass
UCR :: Class Diagram :: Plugin

Description

Each plugin is an instance of this class. Plugin-Authors will be deriving from this class to create plugins. The Gui of the Plugin-Author's script will be the Gui of the plugin (possibly with a bit of "Chrome" around it - plugin tite. delete plugin etc). The Plugin-Author can add Hotkeys and GuiControls to his script to allow the End-User to assign their own hotkeys and tweak the behavior of the script, but otherwise it is a normal AHK script that could do anything.

Attributes

Visibility Name Type Description
public Type String

A Plugin-Author defined unique identifier for this plugin class. At the moment, must be the class name of the Plugin. May be deprecated, if we can find a way of executing a plugin in another thread and discovering the name of the class.
public ParentProfile Profile

A reference to the End-User defined profile that this plugin has been added to.
public Name String

The End-User defined name of the plugin. eg "Remap Joy2 to jump"
public Hotkeys Hotkey

An associative array of Hotkey objects, indexed by name. Every Hotkey declared by the Plugin-Author gets added to this array.
public GuiControls GuiControl

An associative array of GuiControl objects, indexed by name. Every GuiControl declared by the Plugin-Author gets added to this array.

Operations

Visibility Name Description
public AddControl(name: string, ChangeValueCallback: FuncObject, aParams*: variadic)
public AddHotkey(name: string, ChangeValueCallback: FuncObject, ChangeStateCallback: FuncObject, aParams*: variadic)

Plugin-Authors call this to add a Hotkey GuiControl to the plugin. ChangeValueCallback is a callback to be called when the user changes the value of the GuiControl (Chooses a new hotkey). ChangeStateCallback is a callback to be called when the user presses or releases the hotkey. aParams is the params that you would use with the AHK Gui command, skipping param 1, 2 and 3. eg for Gui, Add, Edit, x10 y100, Default You would just omit the Gui param and pass "x10y100", "Default" as aParams.
private _CreateGui()

Creates the GUI for a plugin. Not intended to be used by Plugin-Authors, they use Init()
public Init()

Called when the plugin has done initializing. Does nothing by default - this method is designed to be overridden by the Plugin-Author, to perform any required initialization for their script.
